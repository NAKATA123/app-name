## サービス概要
車修理工場向けの業務管理アプリです。
修理の進捗状況、顧客ごとの修理状況、台車の貸出状況を管理し、スタッフ間で情報を共有することを目的としています。

## このサービスへの思い・作りたい理由
私の親が実際に営んでいる車の修理工場での経験が元になっています。修理工場では、複数の車の修理が同時進行で行われており、「どの車が今どの工程まで進んでいるのか」「どの顧客に代車をいつまで貸しているのか」といった情報が、口頭やホワイトボードで管理されていました。
そのため、スタッフ間での情報共有がうまくいかず、「この修理はもう終わったのか」「この代車は誰が使っているのか」といった確認に時間がかかる場面を何度も見てきました。
こうした課題を解決するために、修理状況・顧客情報・代車の貸出状況を一元管理し、スタッフ全員が同じ情報をリアルタイムで把握できるサービスを作りたいと考え、本サービスを企画しました。

## ユーザー層について
車修理工場のスタッフ・整備士
小規模の修理工場の経営者

## サービスの利用イメージ
ユーザー（修理工場のスタッフ）はログイン後、修理依頼の一覧画面で「どの顧客のどの車が、今どのステータスなのか」を一目で確認できます。
また、顧客詳細画面では、その顧客が現在依頼している修理内容や進捗状況をまとめて確認できます。代車管理画面では、代車が誰にいつからいつまで貸し出されているかを一覧で確認でき、代車の空き状況もすぐに把握できます。
これにより、スタッフ間の情報共有がスムーズになり、確認作業や伝達ミスを減らすことができる価値を提供します。

## ユーザーの獲得について
想定しているユーザー層が小規模な修理工場であるため、口コミや紹介、実際の現場での使用を通じて広がっていくことを想定しています。
また、シンプルな機能構成にすることで、導入しやすいサービスとして展開できると考えています。

## サービスの差別化ポイント・推しポイント
差別化ポイントは、「車修理業務に特化している点」です。
実務に沿った設計を前提に、修理ステータスによる進捗の可視化や
代車の貸出期間管理を組み合わせることで、現場で実際に使いやすい構成にしています。


## 機能候補
MVPリリース時

ユーザー機能
　ユーザー登録 / ログイン / ログアウト
顧客管理
　顧客情報の登録・編集・削除
車両管理
　顧客に紐づく車両情報の登録・管理
修理管理
　修理内容の登録
　修理ステータス管理（受付中 / 作業中 / 完了）
　修理状況の一覧表示
代車管理
　代車の登録
　貸出期間（開始日・終了日）の管理
　代車の貸出状況一覧表示

本リリース

検索機能（顧客名・車両・ステータスなど）
ページネーション
修理履歴の蓄積・過去データの閲覧
ステータスごとの色分け表示などUI改善
権限管理（管理者 / スタッフ）
簡易的なメモ・備考欄機能
